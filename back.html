# 公文润色 Skill

## 功能描述
接收用户提交的公文草稿，自动进行格式规范化、
语言润色和逻辑优化，输出符合政务公文标准的文档。

## 输入
- 公文草稿文本（纯文本或 Markdown）
- 公文类型（通知/报告/请示/批复）

## 输出
- 润色后的公文全文
- 修改批注列表（标注修改原因）
{
  "name": "公文润色",
  "input": {
    "type": "object",
    "properties": {
      "content": { "type": "string", "description": "公文草稿" },
      "doc_type": { "type": "string", "enum": ["通知","报告","请示","批复"] }
    },
    "required": ["content"]
  },
  "output": {
    "type": "object",
    "properties": {
      "polished": { "type": "string" },
      "annotations": { "type": "array" }
    }
  }
}
async def handle(input, context):
    content = input["content"]
    doc_type = input.get("doc_type", "通知")

    analysis = await context.call_tool(
        "text_analyze", {"text": content}
    )

    formatted = await context.call_tool(
        "doc_format", {"text": content, "type": doc_type}
    )

    checked = await context.call_tool(
        "grammar_check", {"text": formatted}
    )

    return {
        "polished": checked["result"],
        "annotations": checked["changes"]
    }
# 公文格式规范

## 标题规范
- 一级标题：方正小标宋简体 2 号
- 二级标题：楷体_GB2312 3 号加粗
- 正文：仿宋_GB2312 3 号

## 排版规则
- 页边距：上 3.7cm、下 3.5cm、左 2.8cm、右 2.6cm
- 行距：固定值 28 磅
- 首行缩进 2 字符

## 常见纠错
- "的"、"地"、"得" 用法校正
- 数字用法（阿拉伯 vs 中文大写）
- 标点符号全角/半角一致性

# 智能翻译 Skill

风险分级管理

管理各 Tool 的风险等级，调整后将在操作日志中记录

P0 高危

不可逆或破坏性操作。执行前强制弹窗二次确认，且记录完整审计日志。

例：文件删除、数据库写入、敏感数据导出

P1 中危

可恢复但有业务影响的操作。执行时Toast 提示用户，并记录操作日志。

例：邮件发送、文件读取、外部 API 调用

P2 低危

只读或无副作用操作。静默执行，仅在执行日志中记录调用信息。

例：语音识别、OCR 识别、文本分类
